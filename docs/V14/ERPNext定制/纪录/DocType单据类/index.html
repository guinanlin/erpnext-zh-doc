DocType单据类

DocType 是任何基于 Frappe 框架的应用程序的核心构建块。

它描述了数据的模型和视图，包含为数据存储字段，以及它们彼此之间的关联关系。它包含如何命名数据的信息，诸如销售订单、销售发票、工单等表单将作为 DocType 存储在数据库。

DocType 允许您根据需要在 ERPNext 中插入自定义表单。

要创建新的 DocType，请转到:

设置 > 自定义 > Doctype > New

1. 1. 如何创建新的 DocType: ​

  1. 输入新 DocType 的名称。
  2. 模块: 输入要添加新 DocType 的模块。
  3. 保存.

1.1 其他详情 ​

  1. 字段

您可以选择添加任意多个字段。标签，字段类型，强制字段和其他相关的选项也可以添加到这里。你可在此了解更多有关字段类型的资料。here.

  1. 命名

    在这里，您可以选择是否希望自动命名此 DocType 中的每个表单。正如在描述中给出的，您可以选择用于命名表单的模式。可以是 字段，命名系列、提示符、已定义的命名系列或基于格式的名称中的字段。对于命名，您还可以添加描述和名称大小写(作为标题大小写或大写大小写) ，以方便使用。

  2. 表单设置

    可以在这里配置表单、图像字段、附件、时间线等的其他设置。要了解有关表单的更多信息，请访问自定义表单。 Customize Form.

  3. 视图

    在这里，您可以为 DocType 定义视图设置，如搜索字段、默认排序字段、默认排序顺序等。

  4. 权限规则

    您可以在此定义 DocType 的权限规则，并配置哪些用户可以使用或更改此 DocType。了解有关用户和权限的更多信息。 Users and Permissions

  5. 网页视图

    您可以选择是否需要此 DocType 的 Web 视图。有一个针对 DocType 的 Web 视图将允许您的网站用户访问表单。请随时了解更多关于网站用户的信息。 Website Users.

1.2. 更多属性 1.2更多物业​

  1. 是可提交的: 您可以通过选中和取消选中此框来选择是否希望此 DocType 仅为“保存”或也为“提交”。

  2. 是子表: 如果希望新建 DocType 是另一个 DocType 中的子表，可以进行定义。有关更多信息，查看子表文档。 Child Table

  3. 是单一的: 如果选中，这个 Doctype 将成为一个单一的表单，如销售订单，用户将无法重新生成。例如，销售模块中的销售设置是单一 DocType。

  4. 是树: ERPNext 中的一些 Doctype 结构为树，其中有一些父 Doctype 和一些子 Doctype。例如，Doctype 公司结构为 Tree，有母公司和子公司，我们称之为子公司。如果希望 DocType 的结构类似，可以启用此选项。

  5. 快速输入: 您可以选择是否要为此 DocType 创建快速输入。这将允许您只输入一些必需的详细信息并保存 DocType，以便创建一个 Quick Entry。例如，选中“日记条目中的快速条目”。 Journal Entry.

  6. 跟踪更改: 如果要维护对每个窗体所做的更改的日志，可以选择此选项。查看文档版本控制以获得对此的更多理解。 Document Versioning

  7. 跟踪可见: 如果您希望维护所有看到此表单的用户的日志，可以选择此选项。

  8. 跟踪视图: 如果要维护每个用户查看此窗体的所有次数的日志，可以选择此选项。

  9. 自定义: 在添加 Custom Doctype 时，默认情况下将检查此字段。类似地，如果您正在自定义系统中已经存在的 DocType，则默认情况下将取消选中此字段
