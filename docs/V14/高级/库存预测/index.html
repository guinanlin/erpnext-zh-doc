库存预测

库存预测是根据当前库存水平和其他需求为特定项目预测的库存水平。

总库存量，包括过去的供应和需求，这是作为计划过程的一部分。

计划库存是用来补货和补充库存的依据。规划引擎使用预计的数量来监视安全库存水平。保持这些水平是为了满足库存的特定需求。

严格控制预计库存对于确定短缺和计算正确的订单数量至关重要。

计算预计库存的数量的公式如下:

Projected Qty = Actual Qty + Planned Qty + Requested Qty + Ordered Qty - Reserved Qty - Reserved Qty for Production - Reserved Qty for Subcontracting - Reserved Qty for Production Plan 预计数量 = 实际数量 + 计划数量 + 请求数量 + 订购数量生产预留数量分包预留数量生产计划预留数量

  • Actual Qty: 实际数量: 仓库中可用的数量。这是您实际拥有的实际库存。
  • Planned Qty: 计划数量: 数量，其中，工作订单已经提出，但正在等待制造。
  • Requested Qty: 请求数量: 通过材料请求请求的数量。它是在提交材料请求时添加的，并在根据材料请求类型创建采购订单/工作订单/库存条目时减去。
  • Ordered Qty: 订购数量: 订购的数量(采购订单) ，但没有收到(通过采购收据或采购发票。
  • Reserved Qty: 保留数量: 客户订购销售的数量(销售订单并且已经提交的) ，但没有交付(通过交货单)。此数量在提交销售订单时增加，而在根据该销售订单创建传递说明或销售发票时减少。
  • Reserved Qty for Production: 生产保留数量: 原材料在提交工作订单时被保留，当原材料通过库存记录转移到在制品仓库时被减少。
  • Reserved Qty for Subcontracting: 分包预留数量: 提交分包采购订单时预留的原材料。当原材料通过库存条目转移到供应商仓库时，这个数量减少了。要了解更多有关分包的信息，请点击这里。
  • Reserved Qty for Production Plan: 生产计划预留数量: 提交生产计划时预留原材料。当物料按照工作订单预留用于生产时，系统将减少相应 BIN 中的“生产计划预留数量”。如果工作订单是按照生产计划进行的，则该系统可以减少数量。
