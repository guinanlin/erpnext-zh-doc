定价规则

定价规则定义了基于设定条件适用的折扣/定价规则。

定价规则有许多选项，您可以使用这些选项来控制物品的定价。可以设置数量、日期、组和其他条件等过滤器。

定价规则与税务规则有些相似。

以下是可以使用定价规则解决的几种情况：

  • 根据促销销售政策，如果客户购买某一物品超过10个单位，他将享受20%的折扣。
  • 对于客户“XYZ”，特定物品的销售价格应更新为###。
  • 属于特定物品组的物品具有相同的销售或购买价格。
  • 属于特定客户组的客户应获得###销售价格，或物品的折扣%。
  • 属于特定供应商组的供应商应应用###的购买率。

要自动应用物品的折扣和价格清单率，请为其创建定价规则。

要访问定价规则列表，请转到：

主页 > 会计 > 定价规则

1. 先决条件​

在创建和使用定价规则之前，建议首先创建以下内容：

  1. 物品
  2. 物品组
  3. 客户
  4. 供应商

2. 如何创建定价规则​

  1. 转到定价规则列表，然后点击新建。
  2. 为规则设置一个标题。
  3. 从物品代码、物品组、品牌或交易中选择要应用的内容。
  4. 选择您想要应用价格折扣或产品折扣。如果要提供免费产品，则选择产品折扣。
  5. 对于单个物品，选择物品代码并选择物品。
  6. 如果要应用定价规则于所有物品，请选择“物品组”并选择所有物品组（父物品组）。
  7. 设置要应用的折扣/价格。要了解更多，请转到此部分。
  8. 保存。

2.1 创建定价规则时的附加选项​

仓库​

在此设置仓库将导致仅当从此处指定的仓库选择物品时才应用定价规则。

应用规则于​

根据“应用于”字段中选择的属性，您可以基于以下之一设置定价规则：

  • 物品
  • 物品组
  • 品牌
  • 交易（对交易总金额）

在此表中，您可以选择特定的物品/物品组/品牌。例如，如果选择“应用于”为“物品组”并在表中选择“原材料”，则此定价规则仅适用于属于“原材料”组的物品。

UoM：仅当此处设置的计量单位与交易匹配时，定价规则才会应用。

条件​

在此字段中，您可以添加一个Python条件，用于检查交易文档类型中的字段值，如下所示，适用于销售发票：

customer=='客户名称' and status!='逾期'

请注意，只有单行Python条件将起作用，使用目标文档类型的字段名。

混合条件​

如果您选择两个或更多物品，并设置最小和最大数量。只有当物品的总和匹配设置的数量时，定价规则才会应用。例如，您在物品1和物品2上创建一个定价规则，并将最小和最大数量设置为30，只有当总数量为30时，定价规则才会应用。

累积的​

启用此选项允许定价规则累积应用。您需要为此设置“最小金额”和“最大金额”。

考虑这样一个情况，最小金额为1,500，最大金额为2,000。现在，如果创建一个金额为1,400的交易，则定价规则不会应用。然而，如果创建一个金额为600的第二张发票，则定价规则将应用。这是因为发票的总金额（累积）达到了2,000。请注意，折扣仅适用于最新的交易，超过累积限制的交易。

这对于如果客户多次购买物品并且您希望以折扣/特价奖励他可能会很有用。

3. 功能​

3.1 在其他项目上应用规则​

此功能在第一个项目上检查条件，但在另一个项目上应用规则。

例如，在“应用规则于”表中设置项目1和项目2，并在项目3上设置“应用规则于其他”。现在，如果交易中包含项目1、项目2和项目3，则定价规则将应用于项目3，因为前两个项目存在于交易中。

3.2 合作伙伴信息​

设置定价规则是用于销售还是采购项目。

根据您的选择，您可以将适用性设置为以下其中一个主要内容：

  • 客户
  • 客户组
  • 地域
  • 销售合作伙伴
  • 营销活动
  • 供应商
  • 供应商组

3.3 数量和金额​

指定项目的最小数量、最大数量、最小金额或最大金额，当应用此定价规则时。

请注意，如果数量或金额低于或超过此处设置的限制，定价规则将根本不会应用。但是，如果您已启用“混合条件”或“累积”选项，则会应用该规则。

3.4 有效期​

您还可以设置定价规则有效的日期间隔。这对于销售促销非常有用。如果将日期留空，则定价规则将没有任何时间范围限制。

3.5 利润​

  • 利润类型：销售项目时，您可以以一定利润出售。如果您不想每次都为项目添加销售价格，并希望自动设置利润，可以使用此功能。
  • 利润率或金额：设置的利润可以基于百分比或金额，例如：5%利润或50美元固定利润。

阅读有关添加利润的更多详细信息。

3.6 价格折扣方案 ​

在这个部分设置要应用的实际规则。

  • 价格: 这将是物品的新价格。例如，如果您以100的价格销售一个物品，并希望以112的价格销售给特定的客户，那么选择价格并将价格设置为112。
  • 折扣百分比: 可以设置特定的折扣百分比。折扣百分比可以设置为特定的价格表。如果将“适用于价格表”留空，则将定价规则应用于所有价格表。
  • 折扣金额: 将应用固定的折扣金额。例如，如果您以100的价格销售一个物品，并希望以7的折扣价格销售，那么可以使用折扣金额选项设置此条件。

3.7 高级设置 ​

  • 建议阈值: 这是基于该阈值，系统将通知您调整物品数量以获得折扣。例如，如果最小数量为10，阈值为9，则系统将通知您添加1个物品以使折扣适用。这也适用于设置的金额。

  • 优先级: 考虑一个物品组，您希望在该组中的一个物品上设置特定规则。可以通过创建一个新的定价规则并设置较高的优先级来实现。这也适用于客户组和供应商组。

  • 应用多个定价规则: 考虑一个价格为500的物品。有两个定价规则P1和P2。P1应用10%的折扣，P2应用5%的折扣。启用此选项将在物品价格上总共应用15%，即425。

  • 在价格上应用折扣: 折扣将被累积。考虑上述相同的情况。启用此选项后，将在500上应用10%的折扣，得到450，然后在450上应用5%的折扣，得到427.5。

  • 验证应用的规则: 如果您在交易中手动设置的折扣/价格与定价规则不匹配，则显示输入的验证消息。

    当层次结构中的顶级分销商决定应用折扣/价格，并且您只验证定价规则是否正确应用时，这将非常有用。

4. 定价规则折扣类型 ​

4.1 价格折扣 ​

  1. 在利润率类型下，您可以设置利润率是按百分比还是按金额计算。例如：销售时按供应商价格表的10%利润率。

  2. 定价规则中提到的价格将优先于物品的价格表（物品价格）价格。

  3. 可以为特定的价格表（销售或采购）应用折扣百分比。要同时应用于两者，请将“适用于价格表”字段留空。

  4. 折扣也可以按金额设置。

4.2 产品折扣 ​

  1. "购买2个数量，获得1个相同物品的免费数量。" 要配置此类规则，请将价格或产品折扣设置为“产品折扣”，选中“相同物品”复选框，并设置数量。

  2. "购买2个数量，获得1个不同物品的免费数量。" 要配置此类规则，请将价格或产品折扣设置为“产品折扣”，取消选中“相同物品”复选框，并设置“免费物品”和数量。
