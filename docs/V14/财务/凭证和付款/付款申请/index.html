付款申请

付款申请用于请求客户为销售订单或发票付款。

付款申请通过电子邮件发送，如果已设置，将包含指向付款网关的链接。您可以通过销售订单或销售发票创建付款申请。还可以针对采购订单或采购发票设置付款申请以供内部记录。然后，可以使用Payment Order批量处理付款。

要访问付款申请，请访问：

主页 > 会计 > 应收账款 > 付款申请

1.先决条件 ​

在创建和使用付款申请之前，建议先创建以下内容：

  1. 销售发票
  2. 采购发票
  3. 销售订单
  4. 采购订单

2. 如何创建付款申请 ​

无法手动创建付款申请，它是根据销售/采购订单或发票创建的。

2.1 通过销售订单创建付款申请 ​

在销售订单中，单击创建，然后单击付款以进行预付款。要了解有关预付款的更多信息，请访问预付款条目页面。

2.2 通过销售发票创建付款申请 ​

在销售发票中，单击创建，然后单击付款以根据发票进行付款。

在支付请求上选择适当的支付网关账户用于账户过帐。在支付网关上指定的帐户负责人将被视为创建日记帐分录。

注意：发票/订单货币和“支付网关账户”货币应该相同。

2.3 通知客户 ​

您可以使用打印格式从付款申请通知客户。如果设置了客户联系电子邮件，则会自动获取。如果不是这样，您可以在付款申请中设置一个电子邮件地址。

2.4 请求邮件 ​

这是一个示例请求电子邮件。如果您已设置相应的支付集成，则会自动生成该 URL。要了解有关集成的更多信息，请访问此页面。

2.5 不使用任何网关的支付请求 ​

如果您不想使用任何集成或支付网关而只想发送通知，只需设置银行账户即可。您必须使用银行详细信息相应地撰写消息。然后，该方可以将金额转入上述银行账户。
